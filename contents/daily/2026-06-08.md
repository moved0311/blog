## Kafka
Topics: consumer可以消耗同topic不同partition的message
Partition: message實際被分配到的地方

1. Producer create and publish a message
   - Message / Record: Key, Value, Timestamp, Headers
2. Kafka assigns the message to the correct topic, broker and partition
3. Message appended to partition via append only log file
4. consumers read next message based on offset

#### when to use kafka ?
- anytime you need a message queue
  - processing can be done async.
- In-order message processing i.e ticket booking
- decouple producer and consumer so they can scale independently ie. Leetcode online judge system
- anytime you need a stream

#### kafka deep dive
1. Scalability
  - Aim for \< 1MB per message
  - one broker up to 1TB data & 10k messages per second
  - choose a good partition key
2. Fault tolerance & durability
3. Error & Retries
4. Performance Optimization
5. Retention Policies

How to handle a hot partition ?
1. remove the key
2. compound key
3. backpressure - slow down the producer
