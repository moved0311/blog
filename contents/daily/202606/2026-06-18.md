## CS50 SQL - Lecture 5 - Optimizing

Indexs:
是找字典如果要找一個單字，如果沒有indexs的話，要從頭一頁一頁翻做比對(SCAN)。但有index的話(像是目錄)可以直接跳到A開頭的字，B開頭的字做查詢(SEARCH)。

PRIMARY KEY會自動做INDEX，但其他columns不會做。

```sql
CREATE INDEX name
ON table (column0, ...);


DROP index name
```

EXPLAIN QUERY PLAN
檢查那些QUERY是SCAN，那些是SEARCH

COVERING INDEX
An index in which queried data can be retrieved from the index itself

一般index搜尋的話假設是(column0有做index)，但是query除了column0可能還要query column1,那這個時候還要回去查column1。這個動作稱做Table Lookup/Back To Table/回表
Covering index是說需要query的欄位本身已經在index中，因為CREATE INDEX可以多個columns

Covering index可以少做一次Table lookup

B-Tree
A balanced tree structure commonly used to create an index

例如title欄位，會先將title做字典排序，之後切分到不同的leaf。
主要節點(trunk node)會記錄每個節點最後一個元素時什麼，當你query時會和節點比較，知道說你查詢的字串是在哪一個節點。
類似binary search，假設你查詢"Toy Story"，主要節點紀錄"Frozen, Soul"，那T在字典排序會在Soul之後，所以會往Soul之後的Leaf查詢。

Vacuum
刪除index對db size有什麼差別
刪除index重新執行`vacuum`才會實際減少db大小

Concurrency
同時間多個query同時發生

```sql
UPDATE "accounts" SET "balance" = "balance" - 10 WHERE "id" = 1;
UPDATE "accounts" SET "balance" = "balance" + 10 WHERE "id" = 2;
```

如果在執行這兩個SQL中斷線，或是有其他人在這兩個SQL間執行查詢，會查詢到錯誤的balance。

Transaction
A unit of work in a database

```sql
BEGIN TRANSACTION;
...
COMMIT;
```

```sql
BEGIN TRANSACTION;
UPDATE "accounts" SET "balance" = "balance" - 10 WHERE "id" = 1;
UPDATE "accounts" SET "balance" = "balance" + 10 WHERE "id" = 2;
COMMIT;
```

ROLLBACK
如果BEGIN TRANSCATION之後的SQL有錯誤，例如balance是負的不符合constrain時，可以執行ROLLBACK將上面的操作REVERT掉，回復到BEGIN TRANSACTION之前

```sql
BEGIN TRANSACTION;
...
ROLLBACK;
```

Race Conditions
用TRANSACTION來避免說類似下面的狀況，
Bob帳戶有$30，Bob同時轉兩次錢給Alice，同時Alice領出$60

| Transfer $30 to A | Transfer $30 to A | withdraw         |
| ----------------- | ----------------- | ---------------- |
| B's balance: $30  |                   |                  |
|                   | B's balance: $30  |                  |
| +$30 to A         |                   |                  |
|                   | +$30 to A         |                  |
|                   |                   | A's balance: $60 |
|                   |                   | withdraw: $60    |
| -$30 from B       |                   |                  |
|                   | -$30 from B       |                  |
|                   | CHCECK FAILED     |                  |
